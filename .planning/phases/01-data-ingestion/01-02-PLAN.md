       ```python
       def build_s3_key(
           source: str,
           entity_type: str,
           year: int,
           month: int,
           day: int,
           filename: str = "data.parquet",
       ) -> str:
           """Build Hive-partitioned S3 key for raw data.

           Format: raw/{source}/{entity_type}/year={y}/month={mm}/day={dd}/{filename}
           Compatible with Athena and dbt external table scanning.
           Zero-padded month and day for correct lexicographic sort order.
           """
           return (
               f"raw/{source}/{entity_type}"
               f"/year={year}/month={month:02d}/day={day:02d}"
               f"/{filename}"
           )
       ```

    4. Define write_parquet_to_s3():
       ```python
       def write_parquet_to_s3(
           records: list[dict],
           schema: pa.Schema,
           bucket: str,
           key: str,
           *,
           region: str = "us-east-1",
       ) -> None:
           """Serialize records to Parquet (snappy) and upload to S3.

           Overwrites any existing S3 object at the given key — idempotent on re-runs.
           Uses BytesIO buffer to avoid writing temp files to disk.
           """
           table = pa.Table.from_pylist(records, schema=schema)
           buf = BytesIO()
           pq.write_table(table, buf, compression="snappy")
           buf.seek(0)
           s3 = boto3.client("s3", region_name=region)
           s3.put_object(
               Bucket=bucket,
               Key=key,
               Body=buf.getvalue(),
               ContentType="application/octet-stream",
           )
           logger.info(
               "parquet_uploaded",
               bucket=bucket,
               key=key,
               row_count=len(records),
           )
       ```

    Critical notes:
    - Use `pa.Table.from_pylist(records, schema=schema)` — the explicit schema= argument is
      MANDATORY to prevent schema drift when records contain None values (Pitfall 4)
    - Pass `region_name=region` to boto3.client so the client connects to the correct endpoint
    - Use `buf.seek(0)` before reading — BytesIO position is at end after write
    - `ContentType="application/octet-stream"` is standard for binary files in S3
    - Do NOT import settings inside this module — callers pass bucket and region explicitly
      so this utility is testable without a real .env file
    - Log at INFO level using structlog after successful upload (not before — avoid logging
      on failures that will be retried)
